================================================================================
 v2.3.0   ―   Cross-device sync (GitHub Gist)
================================================================================

Theme: keep your data in lock-step across every device you use Tasky on.
       Optional, opt-in, offline-first, and entirely under your control.

----------------------------------------------------------------------
[Sync architecture]
----------------------------------------------------------------------
- Single SECRET GitHub gist holds the canonical store JSON
- Each device pulls + merges + pushes
- Per-todo `updatedAt` decides who wins on conflict (last-write-wins)
- Tombstones (store.tombstones.{projects,todos}: { id: deletedAt })
  ensure deletes are honoured across devices and not undone by another
  device pushing an older copy of the same id
- Failures never block local edits; offline use is fully supported

----------------------------------------------------------------------
[Setup flow]
----------------------------------------------------------------------
- New ☁️ Sync header button (or `S` shortcut) opens Sync Settings
- User pastes a GitHub Personal Access Token with `gist` scope
- Tasky calls /user to verify, then /gists to find an existing
  tasky.json gist owned by the user; if none, creates a new private one
- Token + gist id stored ONLY in this browser's localStorage. They
  never leave your device except for talks to api.github.com.

----------------------------------------------------------------------
[Auto sync triggers]
----------------------------------------------------------------------
- Push: 5 seconds after any local change (debounced)
- Pull: on window focus, on `online` event, and every 60 seconds
- Plus a manual "⟳ Sync Now" button in the settings modal

----------------------------------------------------------------------
[Tombstones — the deletion safety net]
----------------------------------------------------------------------
- store.tombstones = { projects: { id: ts }, todos: { id: ts } }
- Recorded by State.deleteProject / deleteTodo
- Cleared by State.undoDelete (so undo also propagates)
- Honoured during merge: if a tombstone is newer than an item's
  updatedAt, the item is dropped from the merge result
- Pruned after 90 days to bound localStorage size

----------------------------------------------------------------------
[Header indicator]
----------------------------------------------------------------------
- ☁️ Sync (grey)              — not configured
- ☁✓ Synced (green)           — last sync OK
- ☁⟳ Syncing… (blue, spinning) — request in flight
- ☁⊘ Offline (amber)          — no network
- ☁⚠ Sync (red)               — last sync failed; tooltip shows error

 v2.0   ―   Phase A · Foundation rebuild
================================================================================

Theme: rebuild the foundation — new data model, modular architecture, and a
       polished UI layer. Every interaction redesigned to be inline and
       non-blocking.

----------------------------------------------------------------------
[Architecture] from monolith to modules
----------------------------------------------------------------------
- 800-line `app.js` class is gone. v2 is split into:
    lib/storage.js    LocalStorage I/O, v1→v2 migration, normalize
    lib/state.js      central state, all mutations, debounced save, pub/sub
    lib/ui-utils.js   toast, confirm modal, save indicator, date helpers
    lib/modals.js     prompt, full-detail todo editor
    lib/render.js     dashboard + project view rendering (no inline onclick)
    lib/actions.js    single delegated handler + drag-drop + inline edit
    lib/keyboard.js   global keyboard shortcuts
    lib/backup.js     JSON / CSV / Markdown export + import + reminder
    lib/help.js       Help modal with 5 tabs + What's New
    app.js            64-line bootstrap
- All state writes go through `state.js`. Saves are debounced 150ms then
  hit localStorage. Renderer subscribes to state changes.

----------------------------------------------------------------------
[Data model] schema v2
----------------------------------------------------------------------
- New top-level: { schemaVersion: 2, projects: [...] }
- Project gets: color (reserved), updatedAt, archivedAt (reserved)
- Todo gets:    description, status (todo/doing/blocked/done — replaces
                bool `completed`), tags[], subtasks[], updatedAt,
                completedAt, order
- Auto-migrate from v1 on load:
    - completed:true  -> status:'done', completedAt set
    - completed:false -> status:'todo'
    - missing fields normalised with defaults
    - v1 localStorage key 'youtodo_projects' kept as backup; v2 writes to
      'youtodo_v2'
- saveSync() used during migration so a quick reload won't re-migrate.

----------------------------------------------------------------------
[Inline editing]
----------------------------------------------------------------------
- Click a task title → it becomes an <input>, focused & selected
- Enter saves, Esc cancels, blur saves
- Click ✏️ button on a task → full editor (description / status / priority
  / due date)
- Click priority badge → cycles Low → Med → High → Low
- Project rename and project title also support inline-style modal prompts.

----------------------------------------------------------------------
[Drag & drop]
----------------------------------------------------------------------
- HTML5 native DnD on .todo-item / .todo-item-mini elements
- Drop on the same list to reorder
- Drop on a different project card (dashboard) to move tasks across projects
  (toast confirmation shown)
- Visual feedback: dragging item dims, target list gets dashed outline,
  target project card gets blue border + slight scale up

----------------------------------------------------------------------
[Toast notifications + Undo]
----------------------------------------------------------------------
- Replaces all alert() / confirm() native dialogs
- Toasts slide in bottom-right; success / warn / error styles
- Delete actions show a toast with `Undo` button (5s timeout)
- Cmd+Z / Ctrl+Z also undoes the most recent delete
- Confirm modal is a custom dialog with Esc/Enter keyboard support

----------------------------------------------------------------------
[Auto-save indicator]
----------------------------------------------------------------------
- New element in the header: "⟳ Saving…" → "✓ Saved" → idle (auto-fades)
- Reflects state.saveStatus updated by the storage debounce flow.

----------------------------------------------------------------------
[Keyboard shortcuts]
----------------------------------------------------------------------
- N            new project
- n            focus task input (project view) or first quick-add (dashboard)
- /            focus the global search box in the header
- 1 - 5        filter All / Active / Completed / Today / Overdue (project view)
- ? or H       open / close Help
- Esc          close any modal · clear search · back to dashboard
- Cmd/Ctrl+Z   undo last delete
- Enter        confirm input / add task / save inline edit
- Cmd+Enter    save in full task editor
Disabled while typing in inputs (except Esc).

----------------------------------------------------------------------
[Filters & search]
----------------------------------------------------------------------
- Project view filter chips: All / Active / Completed / Today / Overdue
- Each chip shows live count
- Global search in the header (1-letter debounce 100ms): matches against
  task text + description + tags

----------------------------------------------------------------------
[Smart due-date labels]
----------------------------------------------------------------------
- Same-day              -> "Today"
- Tomorrow              -> "Tomorrow"
- Yesterday or earlier  -> "Overdue · Aug 5"
- Within 7 days         -> "Mon, Sep 12"
- Further out           -> "Sep 30"
- Overdue rendered red.

----------------------------------------------------------------------
[Help modal + What's New]
----------------------------------------------------------------------
- ❓ Help button in the header (and ? / H shortcut)
- 5 tabs: Overview, Usage, Shortcuts, Backup, What's New
- Click the v2.0 version badge → opens What's New directly

----------------------------------------------------------------------
[Removed inline onclick="..."]
----------------------------------------------------------------------
- Single delegated event handler on #mainContent dispatches by data-action
- Eliminates XSS risk from unescaped data
- Faster repeated rendering (no per-element listener attaching)
- No reliance on a global `app` variable

----------------------------------------------------------------------
[Backup]
----------------------------------------------------------------------
- JSON export now writes the full v2 schema with exportedAt timestamp
- CSV columns expanded: Project, Task, Description, Status, Priority,
  Due Date, Tags (pipe-joined), Created, Completed, Updated
- Markdown report shows tags, status, due dates per task
- Import auto-detects v1 (bare array) vs v2 ({schemaVersion, projects})
- Reminder toast after 7 days of no backup includes a "Backup" button
